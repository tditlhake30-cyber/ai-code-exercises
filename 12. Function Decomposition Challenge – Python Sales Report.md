# Function Decomposition Challenge – Python Sales Report

## Overview

For this exercise I focused on the Python function `generate_sales_report`.  
The function is long and does many different things, which makes it harder to read and maintain.

The goal of the refactoring was to break the function into smaller helper functions with clear purposes, while keeping the same behavior.

---

# Original Problem

The original `generate_sales_report` function is responsible for many tasks at once:

- validating input
- processing date range
- applying filters
- checking for empty results
- calculating summary metrics
- grouping data
- generating detailed report data
- generating forecast data
- generating chart data
- producing output in different formats

Because all of this is inside one function, it becomes difficult to follow.

---

# Responsibilities I Identified

After looking at the function, I identified these distinct responsibilities:

1. Validate the input arguments
2. Filter sales data by date range
3. Apply extra filters
4. Handle the case where no data remains
5. Calculate summary metrics
6. Group data if grouping is requested
7. Build the main report structure
8. Add detailed transaction data
9. Add forecast data
10. Add chart data
11. Generate the final output format

This showed me that the function was doing far more than one job.

---

# Decomposition Plan

My plan was to extract helper functions for the main pieces of logic.

Possible helper functions:

- `validate_report_inputs()`
- `filter_sales_by_date_range()`
- `apply_filters()`
- `build_empty_report_response()`
- `calculate_summary_metrics()`
- `group_sales_data()`
- `build_base_report_data()`
- `build_detailed_transactions()`
- `build_forecast_data()`
- `build_chart_data()`
- `generate_output()`

This makes the main function shorter and easier to understand.

# Refactored Version

```python
from datetime import datetime

def generate_sales_report(
    sales_data,
    report_type='summary',
    date_range=None,
    filters=None,
    grouping=None,
    include_charts=False,
    output_format='pdf'
):
    validate_report_inputs(sales_data, report_type, output_format, date_range)

    if date_range:
        sales_data = filter_sales_by_date_range(sales_data, date_range)

    if filters:
        sales_data = apply_filters(sales_data, filters)

    if not sales_data:
        return build_empty_report_response(report_type, output_format)

    summary = calculate_summary_metrics(sales_data)
    grouped_data = group_sales_data(sales_data, grouping) if grouping else None

    report_data = build_base_report_data(
        report_type=report_type,
        date_range=date_range,
        filters=filters,
        summary=summary
    )

    if grouping:
        report_data['grouping'] = grouped_data

    if report_type == 'detailed':
        report_data['transactions'] = build_detailed_transactions(sales_data)

    if report_type == 'forecast':
        report_data['forecast'] = build_forecast_data(sales_data)

    if include_charts:
        report_data['charts'] = build_chart_data(sales_data, grouping, grouped_data)

    return generate_output(report_data, output_format, include_charts)
```

# Example Helper Functions

```python
def validate_report_inputs(sales_data, report_type, output_format, date_range):
    if not sales_data or not isinstance(sales_data, list):
        raise ValueError("Sales data must be a non-empty list")

    if report_type not in ['summary', 'detailed', 'forecast']:
        raise ValueError("Report type must be 'summary', 'detailed', or 'forecast'")

    if output_format not in ['pdf', 'excel', 'html', 'json']:
        raise ValueError("Output format must be 'pdf', 'excel', 'html', or 'json'")

    if date_range:
        if 'start' not in date_range or 'end' not in date_range:
            raise ValueError("Date range must include 'start' and 'end' dates")

        start_date = datetime.strptime(date_range['start'], '%Y-%m-%d')
        end_date = datetime.strptime(date_range['end'], '%Y-%m-%d')

        if start_date > end_date:
            raise ValueError("Start date cannot be after end date")


def filter_sales_by_date_range(sales_data, date_range):
    start_date = datetime.strptime(date_range['start'], '%Y-%m-%d')
    end_date = datetime.strptime(date_range['end'], '%Y-%m-%d')

    filtered_data = []

    for sale in sales_data:
        sale_date = datetime.strptime(sale['date'], '%Y-%m-%d')
        if start_date <= sale_date <= end_date:
            filtered_data.append(sale)

    return filtered_data


def apply_filters(sales_data, filters):
    filtered = sales_data

    for key, value in filters.items():
        if isinstance(value, list):
            filtered = [sale for sale in filtered if sale.get(key) in value]
        else:
            filtered = [sale for sale in filtered if sale.get(key) == value]

    return filtered


def calculate_summary_metrics(sales_data):
    total_sales = sum(sale['amount'] for sale in sales_data)
    avg_sale = total_sales / len(sales_data)
    max_sale = max(sales_data, key=lambda x: x['amount'])
    min_sale = min(sales_data, key=lambda x: x['amount'])

    return {
        'total_sales': total_sales,
        'transaction_count': len(sales_data),
        'average_sale': avg_sale,
        'max_sale': {
            'amount': max_sale['amount'],
            'date': max_sale['date'],
            'details': max_sale
        },
        'min_sale': {
            'amount': min_sale['amount'],
            'date': min_sale['date'],
            'details': min_sale
        }
    }


def build_base_report_data(report_type, date_range, filters, summary):
    return {
        'report_type': report_type,
        'date_generated': datetime.now().strftime('%Y-%m-%d %H:%M:%S'),
        'date_range': date_range,
        'filters': filters,
        'summary': summary
    }


def generate_output(report_data, output_format, include_charts):
    if output_format == 'json':
        return report_data
    elif output_format == 'html':
        return _generate_html_report(report_data, include_charts)
    elif output_format == 'excel':
        return _generate_excel_report(report_data, include_charts)
    elif output_format == 'pdf':
        return _generate_pdf_report(report_data, include_charts)
```

# Benefits of the Refactoring

Breaking the function down improves the code in a few ways.

### Readability

The main function becomes much shorter and easier to understand because each step has a clear name.

### Maintainability

If one part of the report logic needs to change, it can be updated in a helper function without touching the whole function.

### Testability

Small helper functions are easier to test one by one.

### Reusability

Some helper functions, such as filtering sales data or calculating summary metrics, could be reused in other reporting features.

# Reflection

## How did breaking down the function improve its readability and maintainability?

It made the main function much easier to read because the logic is now split into smaller named steps. Instead of reading one very long function, I can understand the flow more quickly.

## What was the most challenging part of decomposing the function?

The hardest part was deciding where to split the logic without changing the original behavior. Since the function does many related things, it was important not to break the flow incorrectly.

`calculate_summary_metrics()` would probably be the most reusable because summary calculations are common in many reporting features.
