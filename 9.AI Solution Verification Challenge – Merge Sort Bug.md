# AI Solution Verification Challenge – Merge Sort Bug

## Overview

In this exercise I analyzed a sorting algorithm that contains a subtle bug. I used AI to help identify the issue and then applied verification strategies to confirm the solution.

The goal was not just to fix the code, but also to verify the solution carefully and understand why the bug occurred.

---

# Original Buggy Code

```javascript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  return merge(left, right);
}

function merge(left, right) {
  let result = [];
  let i = 0;
  let j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] < right[j]) {
      result.push(left[i]);
      i++;
    } else {
      result.push(right[j]);
      j++;
    }
  }

  while (i < left.length) {
    result.push(left[i]);
    j++; // Bug here
  }

  while (j < right.length) {
    result.push(right[j]);
    j++;
  }

  return result;
}
```

---

# Problem Description

This code implements the **Merge Sort algorithm**, which is used to sort arrays.

The bug occurs in this section:

```
while (i < left.length) {
  result.push(left[i]);
  j++;   // incorrect variable increment
}
```

The code incorrectly increments `j` instead of `i`.

Because of this mistake, the loop may behave incorrectly and could lead to incorrect results or unexpected behavior.

---

# AI Suggested Solution

The AI identified that the incorrect increment variable is the cause of the bug.

The fix is simple:

Change

```
j++
```

to

```
i++
```

This ensures the loop moves through the `left` array correctly.

---

# Verification Strategy 1 – Collaborative Solution Verification

First, I compared the AI suggestion with the algorithm logic.

Merge sort works by merging two sorted arrays together. When one side finishes, the remaining elements from the other side must be added to the result.

Because the loop processes the **left array**, the index `i` must increase.

This confirmed the AI solution was logically correct.

---

# Verification Strategy 2 – Learning Through Alternative Approaches

I looked at other merge sort examples and documentation.

Most implementations use this pattern:

```
while (i < left.length) {
  result.push(left[i]);
  i++;
}
```

This matched the AI suggestion, which gave additional confidence that the fix was correct.

---

# Verification Strategy 3 – Developing a Critical Eye

Instead of blindly trusting the AI, I checked:

• Which array the loop is reading from  
• Which index variable controls that array  
• Whether the increment matches the correct index  

Since the loop uses `left[i]`, the variable `i` must be incremented.

This confirmed the AI solution was correct.

# Final Verified Solution

```javascript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  return merge(left, right);
}

function merge(left, right) {
  let result = [];
  let i = 0;
  let j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] < right[j]) {
      result.push(left[i]);
      i++;
    } else {
      result.push(right[j]);
      j++;
    }
  }

  while (i < left.length) {
    result.push(left[i]);
    i++;   // fixed
  }

  while (j < right.length) {
    result.push(right[j]);
    j++;
  }

  return result;
}
```
