# Error Description

The error message **"IndexError: list index out of range"** means that the program is trying to access a list position that does not exist.

In Python, lists start counting at index **0**.  
If a list contains three items, the valid indexes are:

```
0
1
2
```

In the example above, the loop tries to access index **3**, which does not exist in the list. When Python tries to access that position, it throws an `IndexError`.

---

# Root Cause

The problem is in this line:

```python
for i in range(len(items) + 1):
```

If `len(items)` is **3**, the loop becomes:

```
range(4)
```

Which produces:

```
0, 1, 2, 3
```

But the list only contains indexes:

```
0, 1, 2
```

So when the loop reaches **3**, the program tries to access:

```python
items[3]
```

That index does not exist, which causes the error.

This type of mistake is called an **off-by-one error**, where a loop runs one step too many.

---

# Suggested Solution

The simplest fix is to remove the `+ 1` from the loop.

### Corrected Version

```python
def print_inventory_report(items):
    print("===== INVENTORY REPORT =====")
    for i in range(len(items)):
        print(f"Item {i+1}: {items[i]['name']} - Quantity: {items[i]['quantity']}")
    print("============================")
```

### Better Python Approach

A safer way is to avoid manual indexing and loop directly through the list:

```python
def print_inventory_report(items):
    print("===== INVENTORY REPORT =====")
    for i, item in enumerate(items, start=1):
        print(f"Item {i}: {item['name']} - Quantity: {item['quantity']}")
    print("============================")
```

This method reduces the chance of making indexing mistakes.

### 3. Off-by-one errors are very common

### 5. Iterating directly over objects is safer
Using `enumerate()` or looping over items directly can prevent index mistakes.
