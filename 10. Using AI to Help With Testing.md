# Using AI to Help With Testing

## Overview

In this exercise I used the JavaScript task prioritization algorithm. The main functions involved are:

- calculateTaskScore
- sortTasksByImportance
- getTopPriorityTasks

The goal was to understand what should be tested, improve a test, practice Test Driven Development, and create an integration test.

---

# Part 1 – Understanding What To Test

## Behavior Analysis

The function `calculateTaskScore` calculates a score for tasks based on:

- task priority
- due date
- task status
- special tags
- last update time

### Behaviors to Test

1. Priority affects the base score
2. Tasks due soon or overdue get extra points
3. Completed tasks lose points
4. Tasks with tags like `urgent` or `critical` get a boost
5. Recently updated tasks get extra points

### Edge Cases

- Task with no due date
- Task with unknown priority
- Task already completed
- Task due today
- Task overdue

### Example Test Cases

1. High priority task should have higher score than low priority
2. Task due today should receive extra points
3. Overdue task should get additional score
4. Task with status DONE should lose points
5. Task with `critical` tag should receive bonus points

---

## Test Plan

### Unit Tests

Test each rule individually:

- priority scoring
- due date scoring
- status penalty
- tag bonus
- recent update bonus

### Integration Tests

Test the workflow:

1. calculateTaskScore produces scores
2. sortTasksByImportance sorts tasks correctly
3. getTopPriorityTasks returns the highest scoring tasks

# Part 2 – Improving a Test

## First Simple Test

```javascript
test("calculateTaskScore returns a score", () => {
  const task = {
    priority: TaskPriority.HIGH,
    dueDate: null,
    status: TaskStatus.TODO,
    tags: [],
    updatedAt: new Date().toISOString()
  }

  const score = calculateTaskScore(task)
  expect(score).toBeGreaterThan(0)
})
```

This test is too simple because it only checks that the score is positive.

## Improved Test

```javascript
test("high priority task gets correct base score", () => {
  const task = {
    priority: TaskPriority.HIGH,
    dueDate: null,
    status: TaskStatus.TODO,
    tags: [],
    updatedAt: new Date().toISOString()
  }

  const score = calculateTaskScore(task)
  expect(score).toBe(35)
})
```

This test is clearer because it checks the exact expected score.

## Due Date Test

```javascript
test("task due today gets due date bonus", () => {

    dueDate: today,
    status: TaskStatus.TODO,
    tags: [],
```

# Part 3 – Test Driven Development

## New Feature

Feature: tasks assigned to the current user should receive **+12 score boost**

### First Failing Test

```javascript
test("task assigned to current user gets score boost", () => {
  const task = {
    priority: TaskPriority.LOW,
    assignedToCurrentUser: true,
    dueDate: null,
    status: TaskStatus.TODO,
    tags: [],
    updatedAt: new Date().toISOString()
  }

  const score = calculateTaskScore(task)
  expect(score).toBe(27)
})
```

### Code Update

```javascript
if (task.assignedToCurrentUser) {
  score += 12
}
```

## Bug Fix Test

Bug: incorrect calculation for days since update.

```javascript
test("task updated 3 days ago should not get recent update bonus", () => {
  const oldDate = new Date(Date.now() - 3 * 24 * 60 * 60 * 1000).toISOString()

  const task = {
    priority: TaskPriority.LOW,
    dueDate: null,
    status: TaskStatus.TODO,
    tags: [],
    updatedAt: oldDate
  }

  const score = calculateTaskScore(task)
```

# Part 4 – Integration Test

```javascript
test("task workflow returns highest priority tasks", () => {

  const now = new Date().toISOString()

  const tasks = [
    {
      id: 1,
      priority: TaskPriority.LOW,
      dueDate: null,
      status: TaskStatus.TODO,
      tags: [],
      updatedAt: now
    },
    {
      id: 2,
      priority: TaskPriority.URGENT,
      dueDate: null,
      status: TaskStatus.TODO,
      tags: ["critical"],
      updatedAt: now
    },
    {
      id: 3,
      priority: TaskPriority.MEDIUM,
      dueDate: null,
      status: TaskStatus.DONE,
      tags: [],
      updatedAt: now
    }
  ]

  const sorted = sortTasksByImportance(tasks)
  const topTasks = getTopPriorityTasks(tasks, 2)

  expect(sorted[0].id).toBe(2)
  expect(topTasks.length).toBe(2)
})
```

# Reflection

This exercise helped me understand how to think about testing instead of just writing random tests.

I learned that tests should check specific behavior, not just that the code runs. Writing a simple test first helped me understand the function before improving it.

Test Driven Development also helped me see how writing tests before changing code can guide development.

Integration testing showed how multiple functions should be tested together to make sure the full workflow works correctly.
